# Customer Shopping Behavior Analysis

## Overview

Customer Shopping Behavior Analysis is an end-to-end data analytics project designed to analyze customer purchasing patterns, spending behavior, product preferences, and subscription trends. The project follows the complete analytics workflow from data collection and cleaning to business analysis, dashboard creation, reporting, and presentation.

The objective is to transform raw transactional data into meaningful insights that can help businesses improve customer retention, optimize marketing strategies, and increase revenue.

---

## Dataset

The dataset contains customer transaction records from a retail shopping platform.

### Dataset Summary

* Total Records: 3,900
* Total Features: 18
* Missing Values: 37 values in the Review Rating column

## Project Workflow

### 1. Data Loading

The dataset was imported into Python using Pandas for analysis.

Tasks performed:

* Loaded dataset into a DataFrame
* Examined data structure
* Checked data types
* Generated descriptive statistics

---

### 2. Data Cleaning

Data quality checks were performed to ensure accurate analysis.

Tasks performed:

* Identified missing values
* Imputed missing values in the Review Rating column using the median rating of each product category
* Standardized column names using snake_case naming convention
* Removed redundant columns
* Validated data consistency

---

### 3. Feature Engineering

New features were created to improve analytical capabilities.

Tasks performed:

* Created an `age_group` column by grouping customers into age categories
* Created a `purchase_frequency_days` feature
* Prepared data for customer segmentation analysis

---

### 4. Exploratory Data Analysis (EDA)

Exploratory Data Analysis was performed to understand customer behavior and identify patterns.

Analysis included:

* Customer demographics analysis
* Revenue distribution analysis
* Category-wise sales analysis
* Subscription behavior analysis
* Purchase frequency analysis
* Product rating analysis
* Shipping preference analysis

Visualizations were created using Matplotlib and Seaborn to identify trends and relationships within the data.

---

### 5. Database Integration

After cleaning and transformation, the processed dataset was loaded into a relational database for SQL-based analysis.

Tasks performed:

* Connected Python with PostgreSQL
* Exported cleaned data into database tables
* Prepared data for business query analysis

The SQL queries can also be executed on MySQL and SQL Server with minor syntax modifications.

Determined the most frequently purchased products within each category.

## Key Findings

### Revenue Analysis

* Male customers generated significantly higher revenue than female customers.

### Subscription Analysis

* Non-subscribers represented the majority of customers and generated most of the revenue.
* Average spending between subscribers and non-subscribers was nearly identical.

### Customer Segmentation

* Loyal customers formed the largest customer segment.
* New customers represented only a small portion of the customer base.

### Product Performance

* Gloves, Sandals, Boots, Hat, and Skirt were among the highest-rated products.
* Several products showed strong dependence on discounts for sales.

### Age Group Analysis

* Young Adults generated the highest revenue.
* Revenue contribution was relatively balanced across age groups.

## How to Run

### Clone the Repository

```bash
git clone https://github.com/your-username/customer-shopping-behavior-analysis.git
cd customer-shopping-behavior-analysis
```

### Install Required Libraries


```bash
jupyter notebook
```

Open the notebook and execute all cells.

### Run SQL Analysis

1. Create a database in PostgreSQL, MySQL, or SQL Server.
2. Import the cleaned dataset.
3. Execute the SQL queries available in the SQL folder.

2. Refresh the dataset connection.
3. Explore the dashboard and insights.
